Computational Epidemiology Case Study

Simulating disease spread through matrix-based populations.

SIRSsim is an R package for exploring how local transmission, immunity, mortality, and starting conditions shape SIR, SIS, and SIRS epidemic simulations on two-dimensional grids.

Project Narrative

The package turns a population grid into an analyzable epidemic process.

A user starts with a matrix of susceptible and infected cells, chooses a disease model and probabilities, then runs the simulation until infected cells disappear. The result can be inspected as final state, step history, repeated-run summaries, probability sweeps, heatmaps, or a per-cell event log.

Object of study
Local infection spread on a two-dimensional matrix

Cells are treated as population units. A susceptible cell can become infected when nearby cells are infected, using the probability formula implied by its infected-neighbor count.

Scope
Simulation mechanics, not real-world prediction

The project focuses on controllable SIR-family transition rules, reproducible random seeds, and interpretable outputs rather than fitting a disease to observed population data.

Meaning
Model choices become visible as matrix outcomes

SIR, SIS, SIRS, immunity probability, and optional mortality each change what happens after infection, making assumptions visible at the cell level.

State System

Every simulation is built from four interpretable cell states.

The state encoding is deliberately simple. Matrices use numeric values so the simulation can update, summarize, plot, and log population status without requiring a separate object model.

State encoding
Numeric cell values carry the whole model state.
0 susceptible 1 infected 2 recovered 3 deceased

Initial matrices usually start with susceptible and infected cells. Recovered and deceased states emerge through simulation transitions.

Infection probability
More infected neighbors increase exposure probability. infection_probability = 1 - (1 - prob_infect)^infected_neighbors

A susceptible cell checks its eight-cell neighborhood. If at least one neighbor is infected, the probability compounds by the number of infected neighbors.

Neighborhood test case

prob_infect = 1

The test suite verifies that a single infected center can infect all eight neighboring susceptible cells, while the original infected cell becomes recovered under SIR.

Model transitions

post-infection state
SIR Infected cells become recovered unless optional mortality occurs first.
SIS Infected cells return to susceptible, allowing future reinfection.
SIRS `imm_prob` decides whether infected cells recover; otherwise they return to susceptible.
Mortality `allow_death = TRUE` lets infected cells become deceased with probability `fat_prob` before normal model transition.
Public API

The package separates setup, single runs, and simulation summaries.

The README exposes eight public functions. Together they let a user create starting matrices, run simulations, repeat stochastic trials, sweep infection probabilities, and summarize where recovery occurs.

Matrix setup

Create random matrices with infection probability, exact infected count, and optional seed.

Backward-compatible wrapper around the newer random matrix generator.

Seed patterns

Start infection in the four corners or in the center cell.

Single run

Run SIR, SIS, or SIRS until there are no infected cells left.

Repeated trials

Sweeps and heatmaps

Vary infection probability or count recovered cells across repeated runs.

Outputs

Simulation outputs support both quick summaries and event-level analysis.

The package returns compact result fields by default, then expands to a long-format per-cell log when survival-style analysis is needed. This keeps simple simulations light while preserving a path to deeper endpoint derivation.

Repeated runs
Average stochastic behavior

`simulate_many_runs()` returns total runs, average steps, and average infected proportion for the same starting matrix and probability settings.

Probability sweep

`simulate_inf_seq()` runs simulations across infection probabilities from 0.1 to below 1 using a chosen step size.

Final Project Conclusion

SIRSsim gives users a compact R package for exploring SIR-family disease spread on a grid. Its core contribution is making transition assumptions explicit: who is susceptible, who is infected, what happens after infection, whether mortality is possible, and how repeated stochastic runs change the observed outcome.

The package supports single-run inspection, repeated-run summaries, infection-probability sweeps, recovered-cell heatmaps, and full per-cell logs for survival-style endpoints. That combination lets a new viewer understand the project as both a simulation tool and a teaching surface for epidemic model mechanics.

Recommendations

Present the package as a reproducible simulation surface: start from clear matrix states, choose explicit transition assumptions, inspect both single-run and repeated-run behavior, and use `full_log` when cell-level event timing matters.
